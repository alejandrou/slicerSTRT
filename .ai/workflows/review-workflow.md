# Review Workflow

Review checks whether the implemented task satisfies the approved specification.

## Steps

1. Read `AGENTS.md`.
2. Read the task in `tasks/review/`.
3. Compare the implementation against requirements, out of scope, policies, ADRs, tests, and manual verification evidence.
4. Prioritize bugs, regressions, safety issues, missing validation, and missing tests.
5. Record review findings in the task card.

## Outcomes

- No findings: task may proceed to human approval.
- Findings: task returns to active implementation or backlog, depending on scope.
