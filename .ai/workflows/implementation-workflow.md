# Implementation Workflow

Implementation changes files for the active task only.

## Steps

1. Read `AGENTS.md`.
2. Read the only task in `tasks/active/`.
3. Read required policies, workflows, ADRs, and skills declared by the task.
4. Inspect existing project code before using external examples.
5. Make the smallest change that satisfies the acceptance criteria.
6. Run fast automated checks that do not require unapproved tools or data.
7. Record evidence, changed files, test results, and any skipped checks in the task card or completion report.

## Limits

- Do not modify files outside the task's allowed paths.
- Follow `.ai/policies/git-workflow.md`.
- Follow `.ai/policies/mcp-policy.md`.
- Follow `.ai/policies/dependency-policy.md`.
