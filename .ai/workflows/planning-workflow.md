# Planning Workflow

Planning turns a goal into an approved task card.

## Steps

1. Read `AGENTS.md`.
2. Inspect the active or proposed task card.
3. Inspect only the relevant policies, workflows, ADRs, code, tests, and docs.
4. Confirm whether an ADR is needed for difficult or expensive-to-reverse decisions.
5. Define goal, context, requirements, out of scope, allowed files, skills, acceptance criteria, test plan, manual verification, risks, and documentation impact.
6. Wait for human approval before implementation.

## Output

The planning result is one task card using `.ai/templates/task-template.md`.
